1.Speed up system calls

要求：通过在内核与用户空间之间共享一个只读内存页来加速getpid()系统调用，具体做法是在创建进程时分配一个物理页、存入PID，然后将其以只读方式映射到用户虚拟地址USYSCALL，并在进程退出时释放它，从而让用户程序可以直接从内存读取PID，避免陷入内核。

这个 task 的难点在于，你应该把 map page 这段代码放在哪里最合适？Lab 文档里说的是 When each process is created，具体是什么时候呢？是放在 fork 的地方，还是放在 allocproc 里面，好像都很有道理，但其实这两处都不对。

一开始我决定把代码放在 allocproc 里面，因为 allocproc 对于进程创建都需要调用，甚至第一个进程 /init 也需要调用它，放在这里面可以覆盖到所有用户进程。

前文提到的，用户地址空间的代码、数据都放在 0 开始的一段低地址空间里，而 TRAMPOLINE、TRAPFRAME 以及现在映射的 USYSCALL，都位于高地址空间，在 proc_freepagetable 的时候需要手动释放。

与此同时，我也从 proc_pagetable 和 proc_freepagetable 代码里面学习到了 TRAMPOLINE、TRAPFRAME 的正确映射方式和映射生命周期。所以我们应该把映射 USYSCALL 的代码放在这里面。

2.Print a page table

要求：编写一个名为 vmprint 的函数，用于递归地打印出RISC-V三级页表的详细内容，以便于调试和理解。你需要让这个函数按照指定的格式，通过缩进 .. 来表示页表的层级，并打印出每一个有效的页表项（PTE）的索引、PTE的完整值以及其指向的物理地址。最后，你需要在exec.c中为第一个进程（pid为1）调用vmprint，确保你的输出能通过make grade的pte printout测试。

需要一个 vmprint_raw 真正实现递归遍历打印的辅助函数。同时我记录了一个深度 depth，便于输出 indent。

3.Detect which pages have been accessed

要求：实现一个名为 pgaccess() 的新系统调用，用来检测用户程序的哪些内存页最近被访问过。你需要遍历指定范围的用户虚拟地址，检查每个地址对应页表项（PTE）中的硬件访问位（PTE_A）；如果该位被设置，就将这一信息记录到一个位掩码（bitmask）中，然后必须将该访问位清除，以便将来能检测到新的访问。最后，将这个记录着访问历史的位掩码拷贝回用户空间提供的地址。
